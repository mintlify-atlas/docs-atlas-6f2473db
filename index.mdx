Parlor lets you talk to an AI — and show it your camera — with no cloud, no API keys, and no data leaving your machine. It runs on Apple Silicon and Linux GPUs, achieving real-time end-to-end latency of around 2.5–3 seconds.

## What Parlor does

Parlor streams audio and camera frames from your browser to a local FastAPI server. The server runs **Gemma 4 E2B** (via LiteRT-LM) to transcribe your speech and understand what the camera sees, then streams synthesized speech back using **Kokoro TTS** — sentence by sentence, so you hear the first sentence before the rest is even generated.

## Get started

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/fikrikarim/parlor.git
    cd parlor
    ```
  </Step>
  <Step title="Install uv and sync dependencies">
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    cd src
    uv sync
    ```
  </Step>
  <Step title="Start the server">
    ```bash
    uv run server.py
    ```
    Models download automatically on first run (~2.6 GB for Gemma 4 E2B, plus TTS models).
  </Step>
  <Step title="Open the browser UI">
    Navigate to [http://localhost:8000](http://localhost:8000), grant camera and microphone access, and start talking.
  </Step>
</Steps>

<Note>
  Parlor requires Python 3.12, macOS with Apple Silicon, or Linux with a supported GPU, and approximately 3 GB of free RAM.
</Note>
